IUT Barakah Bites - Testing and Quality Assurance Plan
======================================================

1. Automated Test Matrix:
-------------------------
Every service contains a /tests directory using Jest.
Run all tests:
`npm test` in the root directory.

Key Automated Scenarios:
- Auth: Invalid password, valid token generation, rate limit trigger.
- Order: JWT validation, stock cache miss, idempotency check.
- Stock: Version conflict (optimistic lock), transaction rollback.
- Kitchen: Job processing latency, redis connection resilience.

2. End-to-End Manual Verification (Walkthrough):
----------------------------------------------

Step 1: Database Seeding
- Action: Visit URL /auth/seed.
- Expectation: Database returns code 200 and success message confirming students added.

Step 2: User Login
- Action: Enter student ID into Student UI.
- Expectation: Successful redirect to dashboard; JWT stored in session.

Step 3: Order Placement
- Action: Select item and click 'Purchase'.
- Expectation: Response should be "Order Placed" within < 1 second. Item stock should decrement in backend.

Step 4: Real-time Tracking
- Action: Watch the Order Tracker section.
- Expectation: Status changes from PENDING -> IN_KITCHEN -> READY -> COLLECTED automatically via SSE.

Step 5: Admin Observation
- Action: Open Admin Dashboard.
- Expectation: Health icons turn Green; Latency charts show activity.

3. Resilience & Chaos Testing:
-----------------------------

- Process Termination: Kill a service using 'Chaos Kill' in Admin UI.
  - Expectation: WebApp shows "Offline" alert or visual latency indicator. Remaining services stay functional.
- Database Outage: Disconnect PostgreSQL.
  - Expectation: Health check turns Red. Order Gateway fails gracefully with 502 error.

4. Performance Targets:
----------------------
- ACK Response: All order submissions must return a response in < 500ms.
- UI Load: Frontend must load and become interactive in < 2s on 4G connections.
